What is this tool?

This software helps developers fix issues in cloud systems. It creates a copy of your cloud environment on your local Windows computer. You can test your AI agents and Kubernetes services without needing to upload changes to the cloud server every time. This saves time and prevents errors during development.

✨ Key Software Features

  • Environment Mirroring: Sync your local setup with remote clusters.
  • Traffic Replay: Send real cloud traffic to your local machine for testing.
  • Live DNS Control: Point web traffic to your local agent for debugging.
  • Cloud-Native Support: Built for modern microservices and container tools.

💻 System Requirements

Your computer must meet these basic needs to run the tool without issues:

  • Windows 10 or Windows 11 (64-bit).
  • At least 4GB of RAM.
  • Steady internet connection.
  • Administrator rights to install the network drivers.

📥 Download and Setup

Follow these steps to install the tool on your Windows PC.

  1. Go to the GitHub project page to find the latest version.
  2. Locate the "Releases" section on the right side of the page.
  3. Click the latest release link.
  4. Find the file that ends with .exe and click it to save to your computer.
  5. Find the downloaded file in your Downloads folder.
  6. Double-click the file to start the installer.
  7. Follow the on-screen prompts to complete the setup process.

🚀 Running the Program

Once you finish the setup, open the application from your desktop or start menu.

You will see a main dashboard. Use the navigation buttons on the left to mirror your cloud environment. The tool requires access to your local network. Click Allow when Windows prompts you for network permissions. A green light on the console indicates that the connection is stable.

🛠 Troubleshooting

Check these common fixes if the tool does not work as expected:

  • Check your network connection to the cloud provider.
  • Restart the application as an administrator if features appear greyed out.
  • Ensure your Kubernetes configuration file is in the correct folder path.
  • Update your Windows system if you experience errors during startup.
